Reflect on how confident I am on using the following problem solving techniques:

Pseudocode: Yes, this is very helpful to really understand a problem. It involves writing part code and part 'english' to identify what is required.

Trying something: Yes I had many occasions so far, in which I've tried something and it hasn't worked. I'm very confortable trying out different combinations to try and find the right result.

Rubber ducky method: I haven't used this method yet, but I can see how it will be helpful and I'm sure I will using it in the future.

Reading error messages: I'm starting to get the hang of these. I find it useful to bring up dev tools and try and isolate the issue.

Console.logging: Yes I normally check this out in the dev tools.

Googling: Yes I'm confortable doing this and if often my first point of call.

Asking your peers for help: Yes, happy to ask my peer group. I was probably guilty of going to them too early in the first couple of weeks, without exhausting other options first...must be good for their ego though.

Explain to a non-tech friend what the below functions do.

.map(): This creates a new array with the results of calling a function.

.filter(): This function removes any varibles from an array which do not remain true when the main function is run. For example: The array may hold numbers 1-10 and 15, 17 and 20. The function only runs for numbers 1-10, therefore the filter removes these numbers.

.reduce() This function adds all numbers in the function up and reduces them to one number.
